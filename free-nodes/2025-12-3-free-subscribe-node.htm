一、什么是Policy Quantumult？为何值得使用？

Policy Quantumult，并不是一款单独的App，而是Quantumult或Quantumult X这类高级网络工具中的策略设置模块。它的核心理念是：让不同类型的网络流量走最优路径，并通过策略组、规则分流、协议支持、智能切换等方式，实现用户对整个数据请求过程的控制与调度。

1.1 功能定位

  • 策略定义中心：通过策略组分流、规则优先级判断，实现按需转发流量。

  • 隐私增强工具：允许对敏感请求进行重定向、屏蔽或特定代理。

  • 网络加速助手：依据地域、站点、协议选择最佳服务器路径。

1.2 使用Policy Quantumult的优势

  • 降低延迟：选择距离更近、速度更快的节点自动转发数据。

  • 自由切换代理：为视频、游戏、社交等不同应用配置专属路径。

  • 高度自定义化：黑白名单、域名关键字、正则表达式均支持。

  • 保护隐私安全：防止DNS泄露、提升连接加密等级。

  • 应对封锁与干扰：绕过地域限制、应对复杂防火墙环境。

三、策略配置核心详解：从入门到精通

策略配置是Quantumult的核心。正确理解策略组与规则之间的关系，是构建高效流量管理系统的第一步。

3.1 策略组的类型与作用

策略组类型功能描述
select用户手动切换代理（适用于需要频繁更改）
auto自动选择延迟最低的节点
fallback优先使用主节点，故障时自动切换至备用
url-test定时测速后选择最快服务器

配置示例：

json
[policy_group]
?全球加速 = select, ??香港, ??新加坡, ??美国
?自动代理 = url-test, ??香港, ??日本, ??美国, interval=300

3.2 规则（Rule）的编写

规则是策略的“执行机制”，根据流量类型决定走哪个策略组或节点。

规则类型示例说明
DOMAIN-SUFFIXDOMAIN-SUFFIX, google.com, 全球加速匹配所有以google.com结尾的域名
IP-CIDRIP-CIDR, 192.168.0.0/16, DIRECT匹配局域网地址，直连
GEOIPGEOIP, CN, DIRECT所有中国IP直连
FINALFINAL, 其他策略所有未匹配的请求，走默认规则

规则优先级：规则是自上而下匹配，一旦命中即停止匹配，因此排序很关键。

四、实战配置：一步步搭建你的策略体系

4.1 添加代理节点

  1. 支持手动导入节点（如Shadowsocks/Vmess链接）；

  2. 也可通过订阅链接导入一整套配置；

  3. 节点需命名规范，便于后续引用（如??HK01、??JP02）。

4.2 设定策略组

打开[policy_group]标签，定义所需策略组合，如：

ini
[policy_group]
?全局代理 = select, ??HK01, ??SG01, ??US01
?视频解锁 = url-test, ??Netflix, ??YouTube
?国内直连 = select, DIRECT, ?自动代理

4.3 书写匹配规则

ini
[rule]
DOMAIN-SUFFIX, netflix.com, ?视频解锁
DOMAIN-SUFFIX, youtube.com, ?视频解锁
GEOIP, CN, ?国内直连
FINAL, ?全局代理

4.4 启用配置并测试效果

  1. 应用配置后，访问相关网站测试连接速度；

  2. 在日志/调试界面可实时查看流量走向；

  3. 如遇问题，可使用“诊断模式”检查是否规则设置不当。

  • 使用 doh.google/dns.cloudflare 作为本地DNS；

  • 禁用系统DNS泄露，提高数据隐私性。

六、常见问题答疑（FAQ）

Q1：Policy Quantumult 会影响网速吗？

若配置合理，策略反而会提升整体速度。但若规则复杂或错误可能导致卡顿或无法连接。

Q2：配置失败怎么办？

检查订阅地址、节点可用性，尝试清除缓存或重新导入。

Q3：不同设备之间可以同步配置吗？

Quantumult 不支持云同步，但可通过导出配置文件实现备份与迁移。

Q4：能否设置不同App走不同节点？

iOS系统原生不支持App级别流量分流，但可通过规则匹配其域名间接实现。

Shadowrocket 能否自动更新订阅？
支持。Shadowrocket 可以设置定时更新周期，例如每隔 12 小时或每天自动刷新订阅链接，确保节点列表保持最新状态。这样无需手动更新即可长期稳定使用。
WinXray 的测速功能如何使用？
WinXray 内置节点测速工具，可以通过延迟测试（ping）或带宽测试判断节点质量。用户只需右键节点选择“测速”即可，测试结果将显示在列表中，帮助快速挑选最佳线路。
QuantumultX 的重写脚本可以实现哪些功能？
QuantumultX 支持 JavaScript 重写脚本，可修改请求头、响应内容、跳转 URL、广告屏蔽和定制化流量处理。结合策略组可实现个性化网络访问和优化，提高用户体验。
VMess 协议的动态端口功能是什么？
VMess 支持动态端口（port hopping）功能，能在一定范围内自动变更通信端口。此机制可有效防止防火墙基于端口的封锁策略，提高节点长期稳定性。
VLESS + XTLS 模式适合移动网络吗？
XTLS 模式降低加密开销，减少握手次数，适合高延迟或不稳定移动网络。结合策略组和多出口节点，可提升访问速度和连接稳定性，确保用户体验流畅。
VLESS + XTLS 在高延迟网络中如何优化连接？
XTLS 模式通过减少握手和加密开销降低延迟。结合策略组和多节点切换，可在高延迟或不稳定网络环境下保证网页浏览、视频播放和游戏连接稳定，提高用户体验。
Hiddify Next 如何设置流量配额限制？
管理员可在 Hiddify Next 面板为每个用户设置流量上限或带宽限制。超过配额后可自动阻止或降速，保证服务器资源合理分配，防止单用户占用过多带宽。
Matsuri 节点排序功能如何提升访问效率？
Matsuri 会定期测速节点延迟，并结合策略组自动选择延迟低、稳定性高的节点处理不同流量。提升网页浏览、视频播放和游戏访问速度，减少掉线和卡顿。
V2Ray 支持哪些传输层协议？
V2Ray 支持多种传输层，包括 TCP、mKCP、WebSocket、gRPC、QUIC 等。每种传输方式都有不同的性能特点，例如 TCP 稳定、WS 兼容 CDN、QUIC 延迟低。
